왜 만들었나 — oh-my-cloud-skills

반복되는 고통

Solutions Architect로 일하다 보면 반복적인 작업이 놀라울 정도로 많습니다:

  • EKS 트러블슈팅 — 매번 같은 kubectl 명령어 조합을 검색하고, 같은 진단 절차를 반복
  • 프레젠테이션 제작 — 새 발표마다 HTML 구조를 처음부터 작성
  • 아키텍처 다이어그램 — 비슷한 패턴의 AWS 구성도를 매번 새로 그리기
  • 동료에게 같은 설명 반복 — "이 경우엔 이렇게 하면 돼"를 n번째 설명
핵심 깨달음 처음에는 CLAUDE.md에 자주 쓰는 명령어를 메모하는 것으로 시작했습니다. 그런데 이 "메모장"이 점점 커지면서, 체계적으로 구조화하면 팀 전체가 활용할 수 있겠다는 생각이 들었습니다.

플러그인이라는 해답

Claude Code의 플러그인 시스템은 이 문제를 완벽하게 해결합니다:

  • SA 노하우를 Agent + Skill로 캡슐화
  • /plugin marketplace add 한 줄로 누구나 설치
  • 팀 전체의 생산성 향상 — 개인 지식이 팀 자산으로
  • 지식이 Git에서 버전 관리되며 축적
