Remarp 프레임워크

Remarp는 "Marp를 뒤집은 것"이라는 의미입니다. 마크다운으로 인터랙티브 프레젠테이션을 만드는 차세대 프레임워크로, 이 테크톡의 슬라이드 자체가 Remarp로 제작되었습니다.

Marp의 한계 → Remarp 탄생

기능MarpRemarp
프래그먼트 애니메이션불가{.click} 인라인 프래그먼트
Canvas 애니메이션불가:::canvas DSL
스피커 노트제한적:::notes + 타이밍/큐 마커
인터랙티브 요소불가:::html + :::script

탭/비교/퀴즈불가내장 슬라이드 타입

Remarp 포맷

Remarp는 마크다운에 디렉티브를 추가한 포맷입니다:

---
remarp: true
block: talk
---


@type: cover
@background: ../common/pptx-theme/images/bg.png

# 발표 제목
부제목

---

@type: content

- 두 번째 포인트 {.click}
- 세 번째 포인트 {.click}

Canvas DSL

선언적 문법으로 박스+화살표 애니메이션을 작성합니다:

:::canvas
box api "API Gateway" at 100,180 size 130,55 color #FF9900 step 1
box lambda "Lambda" at 320,180 size 130,55 color #FF9900 step 2
box db "DynamoDB" at 540,180 size 130,55 color #3B48CC step 3
arrow api -> lambda "invoke" step 2
arrow lambda -> db "query" step 3
:::

↑↓ 키로 step을 순차적으로 진행하며 설명할 수 있습니다.

Rejection Loop

LLM은 텍스트 추론에는 뛰어나지만 공간 추론(요소 겹침, 정렬, 레이아웃)은 취약합니다. Rejection Loop는 이 한계를 기계적으로 보완합니다:

# 빌드 전 필수 실행
python3 remarp_to_slides.py validate ./my-presentation/

# 결과 예시
❌ REJECT — 2 CRITICAL issue(s)
  CONTENT_OVERFLOW: Slide 5 has 12 bullet items (max: 8)
  CANVAS_OVERLAP: Slide 8 elements overlap (box A ∩ box B)

7가지 거절 규칙

규칙심각도검사 내용
CONTENT_OVERFLOWCRITICAL불릿 8개 이상 → 슬라이드 분할 필요
CANVAS_OVERLAPCRITICAL캔버스 요소 바운딩 박스 겹침
CANVAS_COMPLEXITYCRITICAL/WARN시각 요소 5+/8+ → HTML 전환 권장
INTERACTIVE_FIRSTWARNING불릿 4+ → 카드/탭 미사용
MISSING_NOTESWARNING:::notes 블록 누락
STATIC_HTMLWARNING:::html 요소 3+ 이나 fragment 없음
FRAGMENT_ORDERWARNING다단 레이아웃에 명시적 order 없음
CRITICAL 이슈가 있으면 빌드 금지 CRITICAL 이슈가 남아있는 상태에서 빌드하면 "프랑켄슈타인 레이아웃"이 생성됩니다. 반드시 수정 후 재검증해야 합니다.

빌드 파이프라인

# 전체 빌드
python3 remarp_to_slides.py build ./my-presentation/

# 특정 블록만 빌드
python3 remarp_to_slides.py build ./my-presentation/ --block 01-intro

# 변경된 블록만 증분 빌드
python3 remarp_to_slides.py sync ./my-presentation/
