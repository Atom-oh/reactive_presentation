Taint/Toleration으로 워크로드 격리

하이브리드 노드에 특정 Pod만 스케줄링

YAML # 노드 Taint 설정 (nodeadm flags) --register-with-taints=eks.amazonaws.com/compute-type=hybrid:NoSchedule # Pod Toleration tolerations: - key: eks.amazonaws.com/compute-type operator: Equal value: hybrid effect: NoSchedule # GPU 노드 추가 Taint --register-with-taints=gpu=true:NoSchedule
핵심: NoSchedule Taint가 있는 노드에는 해당 Toleration이 있는 Pod만 스케줄링됨

규제 대응 & 컴플라이언스

데이터 주권 보장과 감사 추적

데이터 주권 보장

민감 데이터는 온프레미스에 유지, Control Plane만 AWS에서 관리 → 금융 규제 충족

감사 추적

IAM Roles Anywhere의 x509Subject/CN으로 노드별 감사 추적 가능 → 인증서 기반 식별

네트워크 격리

VPN/Direct Connect + Private Endpoint로 퍼블릭 인터넷 경유 없음 → 보안 경계 유지

컴플라이언스 자동화

AWS Config, CloudTrail로 Control Plane 관련 컴플라이언스 자동 모니터링

핵심 가치: 데이터는 온프레미스, 관리는 AWS - 규제와 효율성의 균형

AI Agent 개발 가속화

온프렘 단독 vs EKS Hybrid + AWS 서비스

Before: 온프렘 단독

  • LLM 모델 자체 호스팅 필요 (GPU 인프라 구축)
  • 모델 업데이트/파인튜닝 직접 수행
  • 벡터 DB, 검색 엔진 별도 구축
  • Agent 파이프라인 처음부터 개발
  • 보안/접근 제어 직접 구현
수개월 소요

After: EKS Hybrid + AWS

  • Bedrock API (PrivateLink)로 내부망에서 안전하게 LLM 호출
  • AgentCore로 Agent 오케스트레이션 자동화
  • Knowledge Base + OpenSearch Serverless로 RAG 즉시 구성
  • Neptune Graph DB로 관계 기반 지식 검색
  • Pod Identity 기반 세분화된 IAM 접근 제어
수주 내 PoC 가능
ROI: Bedrock PrivateLink로 내부망 보안 유지 + Pod Identity로 세분화된 접근 제어 — 금융권 요건 충족
