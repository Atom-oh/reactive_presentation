플러그인 아키텍처

디렉토리 구조

모든 플러그인은 동일한 구조를 따릅니다:

plugins/aws-ops-plugin/
├── .claude-plugin/
│   └── plugin.json        # 매니페스트: agents[], skills[], mcpServers{}, hooks{}
├── CLAUDE.md              # 키워드 → 에이전트 라우팅 테이블
├── agents/
│   ├── eks-agent.md       # EKS 전문가 에이전트
│   ├── network-agent.md   # 네트워크 진단 에이전트
│   └── iam-agent.md       # IAM/RBAC 에이전트
└── skills/
    └── ops-troubleshoot/
        ├── SKILL.md        # 진입점 (트리거 키워드 정의)
        └── references/     # 증류된 운영 지식

plugin.json — 매니페스트

플러그인의 모든 구성요소를 선언하는 단일 진실 원천(Single Source of Truth)입니다:

{
  "name": "aws-ops-plugin",
  "version": "1.4.0",
  "description": "AWS infrastructure operations & troubleshooting",
  "agents": [
    "./agents/eks-agent.md",
    "./agents/network-agent.md",
    "./agents/iam-agent.md"
  ],
  "skills": [
    "./skills/ops-troubleshoot",
    "./skills/ops-health-check"
  ],
  "mcpServers": {
    "awsdocs": { "command": "uvx", "args": ["awslabs.aws-documentation-mcp-server@latest"] },
    "awsapi":  { "command": "uvx", "args": ["awslabs.aws-api-mcp-server@latest"] }
  },
  "hooks": { ... }
}

Agent 파일 포맷

에이전트는 마크다운 파일이며, YAML frontmatter에 메타데이터를 선언합니다:

---
name: eks-agent
description: "EKS cluster management, node groups, upgrades, add-ons.
  Triggers on: EKS troubleshoot, cluster issue, node NotReady, pod crash"
tools: Read, Write, Glob, Grep, Bash, AskUserQuestion
---

## Core Capabilities
- Cluster lifecycle management
- Node group troubleshooting (NotReady, OOM, IP exhaustion)
- Add-on version management

## Decision Tree
```mermaid
graph TD
    A[EKS Issue] --> B{Node Level?}
    B -->|Yes| C[kubectl describe node]
    B -->|No| D{Pod Level?}
    D -->|Yes| E[kubectl logs + events]
```

## Error → Solution Table
| Error | Diagnosis | Fix |
|-------|-----------|-----|
| NodeNotReady | kubelet stopped | systemctl restart kubelet |

Skill 파일 포맷

스킬은 디렉토리 단위이며, SKILL.md가 진입점입니다:

---
name: ops-troubleshoot
description: "Systematic EKS/AWS troubleshooting workflow"
triggers:
  - troubleshoot
  - 트러블슈팅
  - diagnose
  - 진단
---

## Workflow
1. Gather symptoms
2. Check cluster health
3. Diagnose root cause
4. Apply fix
5. Verify resolution

references/ 디렉토리에는 공식 문서에서 증류(distill)한 운영 지식을 담습니다. 명령어 중심, Mermaid 의사결정 트리, 에러→솔루션 테이블 형식입니다.

Auto-Invocation (자동 라우팅)

각 플러그인의 CLAUDE.md에 키워드→에이전트 매핑 테이블을 정의합니다:

키워드에이전트
EKS, 클러스터, node NotReady, pod crasheks-agent
VPC, ALB, NLB, DNS, security groupnetwork-agent
IRSA, Pod Identity, RBAC, aws-authiam-agent
프레젠테이션, 슬라이드, 발표자료reactive-presentation-agent

한국어와 영어 키워드를 모두 등록하여 어떤 언어로 질문해도 올바른 에이전트가 활성화됩니다.
