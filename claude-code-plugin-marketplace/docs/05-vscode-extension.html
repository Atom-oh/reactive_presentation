VSCode Extension 개발기

Remarp 마크다운을 작성하고 프리뷰하는 것만으로는 부족했습니다. PPT처럼 시각적으로 편집하고, 리뷰어가 이슈를 남기면 AI가 자동으로 수정하는 워크플로우를 만들고 싶었습니다.

Preview Mode

두 가지 프리뷰 모드를 지원합니다:

모드파일특징
Markdown Preview.remarp.md슬라이드 파싱 → HTML 변환, 사이드바에 노트 + 이슈 배지
HTML Preview빌드된 .html직접 로드, 리소스 경로 → webview URI 변환, CSP 주입

←→ 키보드 네비게이션, Scroll Sync(커서 위치 ↔ 프리뷰 슬라이드 동기화)를 지원합니다.

Visual Edit Mode

Cmd+Shift+E 또는 슬라이드 위 Edit 버튼으로 활성화됩니다:

  • 드래그 — 요소를 마우스로 잡아 위치 이동
  • 리사이즈 — 핸들을 잡아 크기 조절
  • 속성 패널 — 폰트, 색상, 마진 조절
CSS Writeback 비주얼 에디터에서 변경한 내용은 소스 마크다운의 :::css 블록에 자동으로 기록됩니다. Canvas 요소의 경우 :::canvas DSL 좌표가 업데이트됩니다. 소스와 시각적 표현이 항상 동기화됩니다.

Issue Annotation System

이것이 Extension의 가장 핵심적인 기능입니다. 사람과 AI 사이의 피드백 루프를 Extension 안에서 완성합니다:

워크플로우

  1. 이슈 입력 — 사이드바 프롬프트 바에서 "다이어그램이 너무 복잡해" 입력
  2. 자동 삽입 — 소스에 <!-- issue: 다이어그램이 너무 복잡해 --> 자동 추가
  3. 시각화 — 프리뷰에서 노란색 배지로 표시, × 버튼으로 개별 제거 가능
  4. AI 수정 — Claude Code에서 /slide-fix 실행 → AI가 이슈를 읽고 자동 수정
  5. 정리 — 수정 완료 후 <!-- issue: --> 주석 자동 제거
# 이슈 목록 확인
python3 remarp_to_slides.py issues ./my-presentation/

# JSON 형식으로 추출 (프로그래밍적 접근)
python3 remarp_to_slides.py issues ./my-presentation/ --json

주요 파일 구조

tools/remarp-vscode/
├── src/
│   ├── extension.ts       # 진입점: 명령 등록, 파일 감지
│   ├── preview.ts         # 프리뷰 패널: MD/HTML 렌더링
│   ├── htmlPreview.ts     # HTML 전용 프리뷰 핸들러
│   ├── outline.ts         # 슬라이드 아웃라인 프로바이더
│   ├── completions.ts     # 자동완성: @디렉티브, :::블록
│   ├── cssEditor.ts       # CSS 편집: :::css 블록 파싱/업데이트
│   ├── canvasEditor.ts    # Canvas 편집: :::canvas DSL 좌표 업데이트
│   └── visualEditor.ts    # 비주얼 에디터 컨트롤러
├── media/
│   ├── edit-mode.js       # Webview: 드래그/리사이즈/속성 패널
│   ├── canvas-editor.js   # Webview: Canvas SVG 오버레이
│   └── prompt-bar.js      # Webview: AI 프롬프트 바 UI
└── package.json           # Extension 매니페스트
