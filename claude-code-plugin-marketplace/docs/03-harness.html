하네스 엔지니어링

"AI가 잘 달릴 수 있는 레일을 깔아주는 것" — AI 에이전트는 자유도가 높으면 오히려 결과가 불안정합니다. 선언적 제약으로 AI의 행동 범위를 정의하는 것이 핵심입니다.

CLAUDE.md — 컨텍스트 주입

CLAUDE.md는 매 세션 시작 시 자동으로 로드되어 AI에게 프로젝트 컨텍스트를 제공합니다:

  • 프로젝트 구조 — "이 레포에는 3개 플러그인이 있고, 각각 이런 역할을 한다"
  • 빌드/테스트 명령어 — "검증은 이 명령어로 해라"
  • 코딩 컨벤션 — "한/영 이중 키워드를 사용하라"
  • 보안 규칙 — "SG에 0.0.0.0/0 인바운드 절대 금지"
계층 구조 ~/.claude/CLAUDE.md (글로벌 보안 규칙) → 프로젝트/CLAUDE.md (프로젝트 컨벤션) → 플러그인/CLAUDE.md (에이전트 라우팅). 세 계층이 합쳐져서 매 세션에 주입됩니다.

Hooks — 자동 검증

Hooks는 특정 이벤트에 자동 실행되는 셸 명령입니다:

이벤트동작예시
PostToolUse (Bash)빌드 출력에서 경고 패턴 감지remarp_to_slides.py 출력에서 CRITICAL 발견 시 경고
PostToolUse (Edit/Write)파일 변경 시 포맷 검증Remarp frontmatter 누락, 스피커 노트 미작성 감지
SessionStart플러그인 로드 시 도메인 안내"AWS Ops Plugin loaded. Domains: EKS, Network, IAM..."
 
// plugin.json hooks 예시
"hooks": {
  "PostToolUse": [{
    "type": "command",
    "pattern": "Bash",
    "command": "python3 check_build_warnings.py"
  }],
  "SessionStart": [{
    "type": "command",
    "command": "echo 'AWS Ops Plugin loaded.'"
  }]
}

Quality Gate — 품질 보장

모든 콘텐츠는 배포 전에 content-review-agent의 품질 심사를 통과해야 합니다:

판정조건결과
PASSCritical 0, Warning ≤3, Score ≥85배포 승인
REVIEWCritical 0, Warning 4-10, Score 70-84수정 후 재심사
FAILCritical ≥1 or Score <70배포 차단
절대 규칙 Quality Gate를 건너뛰고 배포하는 것은 금지됩니다. 이 규칙은 CLAUDE.md에 명시되어 있어 AI가 자의적으로 무시할 수 없습니다.

세 기둥의 시너지

CLAUDE.md, Hooks, Quality Gate — 이 세 가지 메커니즘의 핵심은 모두 선언적이라는 점입니다:

  • 마크다운과 JSON만으로 AI의 행동 범위를 정의
  • 코드를 작성하지 않아도 제약을 추가/수정 가능
  • 제약 자체가 문서이자 규칙 — 누구나 읽고 이해 가능
